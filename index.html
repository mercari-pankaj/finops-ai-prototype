How Waste Is Estimated

Model over-spec: LiteLLM call logs analyzed for prompt complexity vs. model tier. Calls matching simple patterns (short prompts, structured output) flagged if using Sonnet/Opus tier.

Bot/automation calls: LiteLLM request metadata filtered for non-human callers (scheduled jobs, pipelines). Counterfactual: same call volume at Haiku pricing.

Idle seats: License count cross-referenced against vendor activity data where available. Some vendors do not expose usage-level seat data.

Confidence: High = BigQuery / log-backed. Medium = estimated from partial data or proxy metric. Low = qualitative signal only, not yet measurable.
